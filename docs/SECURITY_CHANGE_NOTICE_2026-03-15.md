# 关于 AI 服务安全配置升级的通知

## 各相关部门及用户：

为进一步加强系统安全防护，防止未经授权的外部访问本地 AI 模型服务，现定于 **2026年3月15日** 对 Memoh 平台的 AI 服务进行安全配置升级。具体事项通知如下：

---

## 一、变更概述

本次升级主要包含以下内容：

1. **启用系统防火墙（UFW）**
   - 默认拒绝所有外部入站连接
   - 仅开放 Memoh Web 界面、API 服务等必要端口

2. **llama-cpp 本地模型服务启用 API Key 认证**
   - 35B 大模型服务（端口 17099）
   - Embedding 向量模型服务（端口 8089）
   - 无 API Key 的请求将被拒绝

3. **新增外部模型支持**
   - 已接入 Kimi Code 模型（仅供 Coding Agents 使用）

---

## 二、影响范围

| 服务 | 影响 | 说明 |
|------|------|------|

## 五、常见问题

**Q1: 不配置 API Key 会怎样？**
> A: 无法使用本地 35B 模型和 Embedding 服务，知识库功能将受影响。

**Q2: 如何申请个人专属 API Key？**
> A: 请联系系统管理员，提供您的用户名和用途说明。

**Q3: 配置后测试连接失败怎么办？**
> A: 请检查：
> 1. API Key 是否输入正确（无多余空格）
> 2. Provider 的 Base URL 是否为 `http://172.17.0.1:17099`
> 3. 如仍有问题，请联系技术支持

**Q4: 外部网络能否直接访问模型服务？**
> A: 不能。17099 和 8089 端口已被防火墙保护，仅允许本地访问。

## 附录：管理员操作参考

### 生成新用户 API Key

```bash
# 生成随机 API Key
openssl rand -hex 32

# 添加到 llama 服务
echo "新KEY" >> /root/.llama/api_keys

# 重启服务生效
systemctl restart llama-qwen35 llama-embedding
```

### 查看当前所有 Key

```bash
cat /root/.llama/api_keys
```

### 删除某个 Key

编辑 `/root/.llama/api_keys` 文件，删除对应行后重启服务。
