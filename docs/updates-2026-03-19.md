## 1. 添加 Kimi Code 模型支持

### 问题
Memoh 无法添加和使用 Kimi Code (kimi-coding) 类型的 Provider，前端添加后无显示。

### 修复文件

| 文件 | 修改内容 |
|------|---------|
| `packages/web/src/data/model-catalog.ts` | 添加 `kimi-coding` provider 配置 |
| `packages/web/src/pages/models/index.vue` | 扩展 `CLIENT_TYPES` 数组 |
| `packages/sdk/src/types.gen.ts` | 更新 `ProvidersClientType` 类型 |
| `packages/cli/src/cli/index.ts` | 更新 CLI choices 数组 |
| `internal/providers/service.go` | 更新 `isValidClientType()` 函数 |
| `agent/src/types/model.ts` | 更新 `SYSTEM_SAFE_PROVIDERS` |

### 使用要点
- Base URL 必须包含 `/v1`，如 `https://api.kimi.com/coding/v1`
- 使用 `kimi-coding` 客户端类型
- 注意：Kimi Code API 对客户端有严格限制，可能无法通过 API 调用

## 2. 修复前端删除模型按钮问题

### 问题
前端删除 LLM 模型的按钮无效，因为传递的是内部 UUID (`model.id`)，但 API 期望的是模型 ID (`model.model_id`)。

### 修复
**文件**: `packages/web/src/pages/models/components/model-item.vue`

```vue
<!-- 修复前 -->
@confirm="$emit('delete', model.id ?? '')"

<!-- 修复后 -->
@confirm="$emit('delete', model.model_id ?? '')"
```

## 4. 优化"思考中"提示动画效果

### 问题
原有的"思考中..."提示是静态文本，用户体验不够生动。

### 改进
添加波浪动画效果，三个点依次变化：
```
⏳ 正在思考中.  →  ⏳ 正在思考中..  →  ⏳ 正在思考中...  →  ⏳ 正在思考中.
```

### 实现
**文件**: `internal/channel/adapters/wecom/adapter.go`

- 每 500 毫秒更新一次动画帧
- 最大持续 30 秒（防止无限循环）
- 添加 ⏳ 沙漏图标提升视觉体验
- 当真实响应准备好时自动覆盖

### 关键代码
```go
// Animation frames for the dots
frames := []string{".", "..", "..."}
ticker := time.NewTicker(500 * time.Millisecond)
```
