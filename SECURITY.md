## 报告漏洞

Memoh-v2 团队非常重视安全问题。如果您发现了安全漏洞，请按照以下步骤报告：

### 报告流程

1. **不要公开披露**：请不要在 GitHub Issues、讨论区或其他公共渠道公开披露安全漏洞。

2. **私下报告**：请通过以下方式之一私下报告：
   - 发送邮件至项目维护者（在 GitHub 个人资料中查找）
   - 使用 GitHub 的 [Private Vulnerability Reporting](https://github.com/91zgaoge/memoh-X/security/advisories/new) 功能（推荐）

3. **提供详细信息**：报告中请包含以下内容：
   - 漏洞的描述和位置
   - 复现漏洞的步骤
   - 可能的影响范围
   - 建议的修复方案（如果有）
   - 您的联系信息（用于后续沟通）

4. **等待回复**：我们会在 48 小时内确认收到报告，并在 7 个工作日内提供初步评估。

### 响应流程

1. **确认收到**：我们收到报告后会立即确认
2. **评估影响**：分析漏洞的严重程度和影响范围
3. **修复开发**：开发修复方案
4. **安全更新**：发布安全更新版本
5. **公开披露**：在修复后负责任地披露漏洞信息（会注明报告者贡献）

## 安全最佳实践

### 部署安全

1. **及时更新**：始终使用最新版本，及时应用安全更新
2. **强密码**：为所有管理账户设置强密码
3. **环境隔离**：生产环境应与开发环境隔离
4. **网络配置**：
   - 仅暴露必要的端口
   - 使用反向代理（如 Nginx）处理 SSL/TLS
   - 配置适当的防火墙规则

### API 密钥管理

1. **安全存储**：不要将 API 密钥硬编码在代码中
2. **最小权限**：为 API 密钥配置最小必要的权限
3. **定期轮换**：定期更换 API 密钥
4. **环境变量**：使用环境变量管理敏感配置

### 容器安全

1. **镜像更新**：定期更新基础镜像
2. **非 root 运行**：尽可能以非 root 用户运行容器
3. **资源限制**：为容器设置 CPU 和内存限制
4. **只读文件系统**：对不需要写入的容器使用只读文件系统

### 数据保护

1. **数据库加密**：对敏感数据进行加密存储
2. **备份加密**：备份文件应加密存储
3. **传输加密**：使用 HTTPS 加密数据传输
4. **定期备份**：定期备份数据并验证可恢复性
